python项目练习十：DIY街机游戏

作者：the5fire | 标签： python实战  python游戏  | 发布：2012-02-09 10:21 p.m. | 阅读量: 67435, 65911

终于来到了最后一个项目，看看前面的那些练习，也算是熟悉了python的基本操作，也知道python能干哪些事情，最后一个项目相比于以前的稍微复杂些，但是任何一个程序只要他是可以正常执行的，花点时间总会搞明白的。

这个练习是一个小游戏程序，如果要是给它起个名字的话，应该叫：快躲，香蕉。主要的游戏内容就是，游戏开始会从屏幕上方不断随便的掉一些铁块，在屏幕下方有一个小香蕉是受你控制的，你需要不断的左右移动来躲避铁块。在你躲避完一定数量的铁块之后，就会进入下一关。下一关依然是让你躲铁块，不过铁块下降的速度就快了很多。在游戏中你可以按下任意键暂停，再次按则继续，按下ESC键退出。这就是全部的功能了，下面我们来看游戏的实现。

无论是在实现功能时还是在代码分析的时候，分类归纳总是一个好习惯，这里自然也不例外。

首先对所有代码分类， 1、整体上代码有一个配置模块，来对游戏的速度、屏幕的宽度、香蕉移动速度、字体大小、各个物体的图片等进行配置。 2、然后是有一个元素模块，即游戏中的两个元素落下来的铁块以及被砸的香蕉，其中还要包含他们具有的行为。 3、然后还有游戏中的各种状态模块，状态模块中的类继承关系稍微多一些，处于家谱最上方的就是state类，由它来衍生其他的所有状态，它的直接子类是Level和Pause，其中Pause有衍生出子类Info、levelCleared、GameOver、StartUp。 4、最后就是游戏的主模块，用来让其他模块协调工作的。

有了上面整体的认识，下面就要细揪一下了。我自己看代码的方法是这样的，首先整体分析，然后在从程序的入口点开始分析。我估计大多数人也是这么做的。

首先是squish.py文件中的game类：

.. code:: python

  1. class Game:
  3.     def __init__(self,*args):
  4.         path = os.path.abspath(args[0])
  5.         dir = os.path.split(path)[0]
  6.         os.chdir(dir)
  7.         self.state = None
  9.         self.nextState = StartUp()
  11.     def run(self):
  12.         pygame.init()
  14.         flag = 0
  16.         if config.full_screen:
  17.                 flag = FULLSCREEN
  18.         screen_size = config.screen_size
  19.         screen = pygame.display.set_mode(screen_size,flag)
  21.         pygame.display.set_caption('Fruit Self Defense')
  22.         pygame.mouse.set_visible(False)
  24.         while True:
  25.             if self.state != self.nextState:
  26.                 self.state = self.nextState
  27.                 self.state.firstDisplay(screen)
  29.             for event in pygame.event.get():
  30.                 self.state.handle(event)
  31.             self.state.update(self)
  33.             self.state.display(screen)
  35. if __name__ == '__main__':
  36.     game = Game(*sys.argv)
  37.     game.run()

忽略掉init中的设置代码，在run中，该管理类首先调用pygame初始化并启动游戏界面，然后在一个while True的死循环中不断的进行状态判断，事件处理，然后根据事件更新当前状态，并且绘制界面。

让我们把焦点放在那个死循环中，因为他就是整个程序的流程所在。 其中状态和事件的关系就是,当发生某一事件之后，状态就会发生变化，比如点击事件、过关事件、死亡事件。这些事件的来源分别是：用户操作、系统判断、系统判断。要继续深入分析就需要再拿一部分代码出来。

依然是来自squish.py文件中剩余的所有代码：

.. code:: python

  1. import os, sys, pygame
  2. from pygame.locals import *
  3. import objects, config
  5. class State:
  6.     def handle(self,event):
  7.         if event.type == QUIT:
  8.             sys.exit()
  9.         if event.type == KEYDOWN and event.key == K_ESCAPE:
  10.             sys.exit()
  12.     def firstDisplay(self, screen):
  13.         screen.fill(config.background_color)
  14.         pygame.display.flip()
  16.     def display(self, screen):
  17.         pass
  19. class Level(State):
  21.     def __init__(self,number=1):
  22.         self.number = number
  23.         self.remaining = config.weights_per_level
  25.         speed = config.drop_speed
  27.         speed += (self.number - 1) * config.speed_increase
  29.         self.weight = objects.Weight(speed)
  30.         self.banana = objects.Banana()
  31.         both = self.weight,self.banana
  32.         self.sprites = pygame.sprite.RenderUpdates(both)
  34.     def update(self, game):
  35.         self.sprites.update()
  37.         if self.banana.touches(self.weight):
  38.             game.nextState = GameOver()
  40.         elif self.weight.landed:
  41.             self.weight.reset()
  42.             self.remaining -= 1
  43.             if self.remaining == 0:
  44.                 game.nextState = LevelCleared(self.number)
  46.     def display(self, screen):
  47.         screen.fill(config.background_color)
  48.         updates = self.sprites.draw(screen)
  49.         pygame.display.update(updates)
  51. class Paused(State ):
  52.     finished = 0
  53.     image = None
  54.     text = ''
  56.     def handle(self, event):
  57.         State.handle(self, event)
  58.         if event.type in [MOUSEBUTTONDOWN,KEYDOWN]:
  59.             self.finished = 1
  61.     def update(self, game):
  62.         if self.finished:
  63.             game.nextState = self.nextState()
  65.     def firstDisplay(self, screen):
  66.         screen.fill(config.background_color)
  68.         font = pygame.font.Font(None, config.font_size)
  69.         lines = self.text.strip().splitlines()
  71.         height = len(lines) * font.get_linesize()
  73.         center,top = screen.get_rect().center
  74.         top -= height // 2
  76.         if self.image:
  77.             image = pygame.image.load(self.image).convert()
  79.             r = image.get_rect()
  80.             top += r.height // 2
  82.             r.midbottom = center, top -20
  84.             screen.blit(image, r)
  86.         antialias = 1
  87.         black = 0,0,0
  90.         for line in lines:
  91.             text = font.render(line.strip(),antialias,black)
  92.             r = text.get_rect()
  93.             r.midtop = center,top
  94.             screen.blit(text, r)
  95.             top += font.get_linesize()
  97.         pygame.display.flip()
  99. class Info(Paused):
  101.     nextState = Level
  102.     text = '''
  103.     In this game you are a banana,
  104.     trying to survive a course in
  105.     self-defense against fruit,where the 
  106.     participants will 'defend' themselves
  107.     against you with a 16 ton weight.'''
  109. class StartUp(Paused):
  111.     nextState = Info
  112.     image = config.splash_image
  113.     text = '''
  114.     Welcome to Squish.
  115.     the game of Fruit Self-Defense'''
  117. class LevelCleared(Paused):
  119.     def __init__(self, number):
  120.             self.number = number
  121.             self.text = '''Level %i cleared
  122.             Click to start next level''' % self.number
  124.     def nextState(self):
  125.             return Level(self.number + 1)
  127. class GameOver(Paused):
  128.     nextState = Level
  129.     text = '''
  130.     Game Over
  131.     Click to Restart, Esc to Quit'''

其中用户判断部分就是Paused类中的update方法和handle方法，而系统判断就是Level类中的update方法。还有一个要注意的地方就是Level类中update方法中的第一行代码：self.sprites.update()，这是让铁块不断下落的关键代码。用户判断部分的代码已经有了，下面需要贴上系统判断时用到的代码.

objects.py中的代码：

.. code:: python

  1. import pygame,config,os
  2. from random import randrange
  4. class SquishSprite(pygame.sprite.Sprite):
  6.     def __init__(self, image):
  7.         pygame.sprite.Sprite.__init__(self)
  8.         self.image = pygame.image.load(image).convert()
  9.         self.rect = self.image.get_rect()
  10.         screen = pygame.display.get_surface()
  11.         shrink = -config.margin*2
  12.         self.area = screen.get_rect().inflate(shrink,shrink)
  14. class Weight(SquishSprite):
  16.     def __init__(self, speed):
  17.         SquishSprite.__init__(self,config.weight_image)
  18.         self.speed = speed
  19.         self.reset()
  21.     def reset(self):
  22.         x = randrange(self.area.left, self.area.right)
  23.         self.rect.midbottom = x, 0
  25.     def update(self):
  26.         self.rect.top += self.speed
  27.         self.landed = self.rect.top >= self.area.bottom
  29. class Banana(SquishSprite):
  31.     def __init__(self):
  32.         SquishSprite.__init__(self, config.banana_image)
  33.         self.rect.bottom = self.area.bottom
  35.         self.pad_top = config.banana_pad_top
  36.         self.pad_side = config.banana_pad_side
  38.     def update(self):
  39.         self.rect.centerx = pygame.mouse.get_pos()[0]
  40.         self.rect = self.rect.clamp(self.area)
  42.     def touches(self, other):
  43.         bounds = self.rect.inflate(-self.pad_side,-self.pad_top)
  45.         bounds.bottom = self.rect.bottom
  47.         return bounds.colliderect(other.rect)

在类Banana和Weight中的update和touches方法，用于进行系统判断。

好了，到这主要的东西都分析完了，剩下的只需要稍看一下就能够懂得了。
